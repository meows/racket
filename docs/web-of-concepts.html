Web of Concepts

This is a document of essential terminology that I will be including as part of this Racket tutorial series. It will describe the minimal set of foreign but also formal terms that I assess to be the "damage" of learning some minimum fluent corpus of Racket.

Symbol

A symbol is a discrete character which can be used as the basis for building arbitrary sequences of symbols.

Alphabet

The finite set of legitimate symbols is called an alphabet.

Atom

An atom is a finite sequence of symbols. These sequence of symbols are very flexible, and can be used as both a source of names and values. But an atom provides a property to both names and values such that they aren't to be divided any further. For example,

The name add3 isn't meant to be divided further into: add and 3, or some other combination. It's one whole name.

The value of 199.99 isn't meant to be divided any further into 19 and 9.99. It's one whole value.

This point might seem strange, but when we start staring at what we write in an abstract way, as if looking at a foreign language, we might be forgiven into mis-dividing a German word. Plus, sometimes we chop up mathematical atoms and fruitful insights come out.

The atom split in two might render two new sensible atoms, but in doing so we have violated the spirit of an atom, whose very constraints permits us to communicate to both human and machine what is a singular thing.

It's not as though the interpreter, or whatever consumes our program, must necessarily submit to the tyranny of our instructions, but we are warning that in doing so, they are likely going where we have failed to encode information.

Lastly, atoms also allow us to efficiently reuse a limited alphabet. It's strange to think that aside from a few funny characters, everything else on this page is constructed from 26 symbols, known as letters. If we had an infinite amount of symbols, we might not need atoms.

Value

A value is any piece of information encoded using an atom, but a separate term is warranted because atoms are used to generate values. The difference between a value and a name is that variations in a name are generally meant to point to different places and be meaningful only to humans, whereas variations in a value are meant to be potentially meaningful down to every granularity of variation.

Expression

An expression is either an atom or a list which is reducible to an atom. This means that an expression can hold more expressions, contextually called sub-expressions.

List

A list is a sequence of zero or more expressions. A list surrounds its members with a pair of brackets, and its members are separated by spaces.
